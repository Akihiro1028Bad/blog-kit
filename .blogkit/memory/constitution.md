## Writing Principles

### Clarity First
- MUST write in clear, concise language
- MUST avoid unnecessary jargon
- MUST explain technical terms when first introduced

### Structure Standards
- MUST use consistent heading hierarchy
- MUST include clear introduction and conclusion
- MUST organize content logically

### Code Examples
- MUST include working code examples when explaining technical concepts
- MUST add comments to explain complex code
- MUST test all code examples before including

### Quality Standards
- MUST proofread before publishing
- MUST check for broken links
- MUST verify all technical information

## Content Guidelines

### Target Audience
- Define target audience clearly in spec.md
- Write at appropriate technical level
- Provide context for beginners when needed

### Length Guidelines
- Introduction: 300-500 words
- Main sections: 1000-2000 words each
- Conclusion: 300-500 words
- Total: Adjust based on topic complexity

### Visual Elements
- Use diagrams when they add clarity
- Include screenshots for UI-related content
- Add code snippets with syntax highlighting

## Technical Requirements

### Markdown Format
- Use standard Markdown syntax
- Include frontmatter with metadata
- Use consistent code block formatting

### File Organization
- One article per directory
- Store assets in `assets/` subdirectory
- Keep spec.md, plan.md, tasks.md for reference
